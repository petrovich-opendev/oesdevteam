# External Data Resilience Rules — Cross-Language

> Canonical rule set for code that consumes messages from external
> systems: brokers (Kafka / NATS / MQTT / RabbitMQ), SCADA, FMS, OPC-UA,
> ModBus, vendor telemetry HTTP APIs. **Referenced by every Senior
> Reviewer prompt** — Senior Backend, Senior Go, Senior Data, Senior
> SRE, Senior Frontend (when consuming WebSocket / SSE / push). Rule
> IDs (R-1 … R-12) are stable and are cited verbatim in reviewer
> findings; do not renumber.

This document is **language-neutral**: each rule states the principle
once, then shows a side-by-side **Python**, **Go**, and **TypeScript**
implementation. A reviewer who flags a rule cites the rule ID; the
authoring developer reads the example matching their language.

## Why this file exists

Industrial data sources are hostile by nature. A consumer that assumes
well-formed input breaks in production within days:

- A SCADA engineer renames a tag; every downstream consumer crashes.
- A firmware upgrade reorders JSON keys; the schema "changes" without
  a version bump.
- A broker reconnect delivers a half-written message; the parser
  explodes.
- A network blip creates 30 s of backpressure; the consumer OOMs.
- A vendor starts emitting timestamps in a different timezone; all
  downstream analytics shift by 3 h silently.

Our rule: **no single malformed message, renamed tag, or transient
connection blip may take the service down.** A consumer MUST drop and
keep going; it MUST NOT crash, retry forever, or silently swallow.
Worst allowed outcome is "drop the message, increment a counter,
surface via health check, keep consuming". A pager at 03:00 for a
Kafka hiccup is a bug.

## Language scope

| Lang | Primary use                                     | Notes                                     |
| ---- | ----------------------------------------------- | ----------------------------------------- |
| Python | ETL / analytics / legacy stack                | Pydantic + asyncio                        |
| Go   | Industrial Go service mesh, broker consumers    | pgx + slog + OTel + chi/grpc              |
| TS   | Browser / Node consumers of push channels       | Zod for schemas; many rules N/A on browser |

Each rule's "TS" block is omitted (with explicit "N/A — backend rule")

## Rule R-1: Validate at the boundary, degrade gracefully

**Every incoming message MUST be parsed through a typed schema at the
consumer entry point.** No downstream business logic may ever see a
raw dict / `map[string]interface{}` / `unknown`. The parser failure
MUST be caught at exactly one place and produce a drop, not an
exception that unwinds the worker.

**Severity if missing: BLOCKER.** A consumer that propagates the
parser exception to the event loop will crash a worker on the first
firmware upgrade.

### Python

```python
async def on_message(raw: bytes) -> None:
    try:
        msg = IncomingEvent.model_validate_json(raw)
    except (json.JSONDecodeError, ValidationError) as e:
        DROPPED_MESSAGES.labels(reason="invalid_schema").inc()
        logger.warning("dropped malformed message: %s", _truncate(e))
        return
    await handle(msg)
```

### Go

```go
func (c *Consumer) onMessage(ctx context.Context, raw []byte) {
    var msg IncomingEvent
    if err := json.Unmarshal(raw, &msg); err != nil {
        c.dropped.WithLabelValues("invalid_schema").Inc()
        slog.Warn("dropped malformed message", "err", err, "len", len(raw))
        return
    }
    if err := msg.Validate(); err != nil { // generated by sqlc/protovalidate
        c.dropped.WithLabelValues("invalid_schema").Inc()
        slog.Warn("dropped invalid payload", "err", err)
        return
    }
    c.handle(ctx, msg)
}
```

### TypeScript

```ts
import { z } from "zod";

const IncomingEvent = z.object({
  ts: z.string().datetime(),
  deviceId: z.string(),
  value: z.number(),
});

export function onMessage(raw: string) {
  const parsed = IncomingEvent.safeParse(JSON.parse(raw));
  if (!parsed.success) {
    metrics.dropped.inc({ reason: "invalid_schema" });
    log.warn("dropped malformed message", { issues: parsed.error.issues });
    return;
  }
  handle(parsed.data);
}
```

## Rule R-2: Drop, count, log — never crash, never silently swallow

For every drop path, all three MUST be present:

1. **Drop** — the consumer moves on. No retry-forever, no DLQ-into-self.
2. **Count** — a counter increments labelled by reason
   (`invalid_schema`, `unknown_tag`, `duplicate`, `backpressure`,
   `timestamp_drift`, etc.).
3. **Log** — one WARN line with truncated payload excerpt (no full
   payloads — PII risk). First-seen-per-reason at ERROR.

Silent drop = **BLOCKER**. Drop without counter = **MAJOR**. Drop
without log = **MAJOR**. All three together = **PASS**.

### Python

```python
DROPPED_MESSAGES.labels(reason="unknown_tag").inc()
logger.warning("dropped: unknown tag %s", _truncate(tag_name))
return
```

### Go

```go
c.dropped.WithLabelValues("unknown_tag").Inc()
slog.Warn("dropped: unknown tag", "tag", truncate(tagName))
return
```

Note: in Go, the engine pre-review rule `R-go-silent-err` (BLOCKER)
specifically catches the silent-drop-on-error case (`if err != nil
{ }` / `_ = err` / bare `return nil`). A silent drop in the *control
flow* (no error involved) is reviewer-enforced under R-2 here.

### TypeScript

```ts
metrics.dropped.inc({ reason: "unknown_tag" });
log.warn("dropped: unknown tag", { tag: truncate(tagName) });
return;
```

## Rule R-3: Health check reflects drop rate

A health endpoint that returns `200 OK` while 95 % of messages are
being dropped is lying to the operator. The health check MUST surface
the recent drop fraction and `last_successful_receive` timestamp.

In Go services, R-3 lives on `/readyz` (the readiness probe) — not on
`/livez`. Drop rate is a *traffic-readiness* concern, not a
process-alive concern.

**Severity if missing: MAJOR** (consumer is live but invisible to
on-call); **BLOCKER** if the service is declared production-ready.

### Python

```python
@app.get("/health")
async def health() -> HealthResponse:
    received = RECEIVED_MESSAGES.recent_rate(window_seconds=60)
    dropped = DROPPED_MESSAGES.recent_rate(window_seconds=60)
    drop_fraction = dropped / received if received else 0.0
    status = "ok" if drop_fraction < 0.10 else "degraded"
    if drop_fraction > 0.50:
        status = "down"
    return HealthResponse(
        status=status,
        drop_fraction=drop_fraction,
        last_successful_receive=LAST_SUCCESS.timestamp(),
    )
```

### Go

```go
func (h *Health) ReadyZ(w http.ResponseWriter, r *http.Request) {
    received := h.received.Rate(60 * time.Second)
    dropped := h.dropped.Rate(60 * time.Second)
    dropFraction := 0.0
    if received > 0 {
        dropFraction = dropped / received
    }
    status := "ok"
    code := http.StatusOK
    switch {
    case dropFraction > 0.50:
        status, code = "down", http.StatusServiceUnavailable
    case dropFraction > 0.10:
        status = "degraded" // still 200 — degraded but serving
    }
    writeJSON(w, code, readyzResponse{
        Status:                  status,
        DropFraction:            dropFraction,
        LastSuccessfulReceive:   h.lastSuccess.Load(),
    })
}
```

### TypeScript

N/A — backend rule. Browser clients do not expose a readiness probe.

## Rule R-4: Schema evolution is versioned, renames are explicit

SCADA / FMS tag renames are not a storm event — they happen on every
control-system upgrade. Code defends structurally.

**Two viable patterns:**

**Pattern A — Envelope with explicit version:** dispatch on
`schema_version`; unknown versions drop with reason
`unknown_schema_version`.

**Pattern B — Field-mapping config (preferred for SCADA / FMS):**

```yaml
# config/tag_mappings.yaml
internal_to_external:
  probe_temp_c:
    primary: "P_Temp_1"               # current vendor name
    aliases: ["Probe.Temp.1", "PT1"]   # accepted historical names
    required: true
    unit: "celsius"
```

Loader reads each incoming message, looks up each field via the
mapping, accepts aliases, rejects missing required tags with reason
`missing_required_tag:probe_temp_c`.

**Severity if vendor tag-name is hardcoded in source: BLOCKER** for
any SCADA / FMS / vendor-telemetry integration. The next firmware
rollout will take the consumer down.

### Python

```python
class TagMapping(BaseModel):
    primary: str
    aliases: list[str] = Field(default_factory=list)
    required: bool = True
    unit: str | None = None

def resolve(name: str, mapping: dict[str, TagMapping]) -> str | None:
    for internal, m in mapping.items():
        if name == m.primary or name in m.aliases:
            return internal
    return None
```

### Go

```go
type TagMapping struct {
    Primary  string   `yaml:"primary"`
    Aliases  []string `yaml:"aliases"`
    Required bool     `yaml:"required"`
    Unit     string   `yaml:"unit"`
}

// Resolve maps a vendor tag name to the project's internal field name.
// Returns "" when the tag is unknown — the caller drops with reason
// "unknown_tag" (R-2).
func (m TagMappings) Resolve(name string) string {
    for internal, def := range m {
        if name == def.Primary {
            return internal
        }
        for _, alias := range def.Aliases {
            if name == alias {
                return internal
            }
        }
    }
    return ""
}
```

N/A — backend rule (vendor tag mapping is a backend concern; browser
clients consume already-normalised events).

## Rule R-5: Unknown / extra fields tolerated by default

For evolving external schemas, the parser default tolerates unknown
fields. A new vendor field must not crash a consumer that doesn't use
it yet.

**Severity if a strict-mode parser is set on external messages:
MAJOR** unless there is a written justification (the message is from a
system under our control and contract violations must fail loud).

### Python

```python
class TelemetryEvent(BaseModel):
    model_config = ConfigDict(extra="ignore")  # forward-compatible
    ts: datetime
    device_id: str
    value: float
```

### Go

```go
// Go's encoding/json ignores unknown fields by DEFAULT. Do NOT call
// dec.DisallowUnknownFields() on vendor payloads — that ships
// fragility (rule R-5 violation, MAJOR).
type TelemetryEvent struct {
    Ts       time.Time `json:"ts"`
    DeviceID string    `json:"device_id"`
    Value    float64   `json:"value"`
}

dec := json.NewDecoder(r)
// dec.DisallowUnknownFields()  // ← would be R-5 violation
if err := dec.Decode(&ev); err != nil { ... }
```

### TypeScript

```ts
// Zod's default is .strict() === false → unknown fields ignored.
// Calling .strict() on a vendor schema is the R-5 violation.
const TelemetryEvent = z.object({
  ts: z.string().datetime(),
  deviceId: z.string(),
  value: z.number(),
}); // do NOT add .strict()
```

## Rule R-6: Connection faults use bounded retry + circuit breaker

Every broker / SCADA / vendor client implements:

1. Bounded reconnect (exponential backoff, cap ≤ 30 s).
2. Circuit-breaker state: `closed` → `open` after N consecutive
   failures; `half_open` probe; back to `closed` on success.
3. State visible on the readiness endpoint.

Unbounded `for { connect() }` = **BLOCKER**.

### Python

```python
class BrokerClient:
    """Connection state is visible to the health endpoint."""

    state: Literal["closed", "half_open", "open"] = "closed"
    last_error: str | None = None
    last_success_ts: float = 0.0

    async def connect_with_backoff(self) -> None:
        delay = 0.5
        consecutive = 0
        while not self._stopped:
            try:
                await self._connect()
                self.state = "closed"
                self.last_success_ts = time.time()
                return
            except BrokerError as e:
                self.last_error = str(e)
                consecutive += 1
                if consecutive >= 5:
                    self.state = "open"
                BROKER_RECONNECTS.inc()
                await asyncio.sleep(min(delay, 30.0))
                delay = min(delay * 2, 30.0)
```

### Go

```go
type BrokerClient struct {
    state       atomic.Int32 // 0=closed, 1=half_open, 2=open
    lastErr     atomic.Pointer[string]
    lastSuccess atomic.Int64 // unix nanos
    reconnects  prometheus.Counter
}

func (c *BrokerClient) ConnectWithBackoff(ctx context.Context) error {
    delay := 500 * time.Millisecond
    consecutive := 0
    for {
        if err := ctx.Err(); err != nil {
            return err
        }
        if err := c.connect(ctx); err != nil {
            consecutive++
            s := err.Error()
            c.lastErr.Store(&s)
            if consecutive >= 5 {
                c.state.Store(2) // open
            }
            c.reconnects.Inc()
            slog.Warn("broker reconnect", "err", err, "delay", delay)
            select {
            case <-ctx.Done():
                return ctx.Err()
            case <-time.After(delay):
            }
            delay = minDuration(delay*2, 30*time.Second)
            continue
        }
        c.state.Store(0) // closed
        c.lastSuccess.Store(time.Now().UnixNano())
        return nil
    }
}
```

### TypeScript

```ts
// Browser WebSocket reconnect with bounded backoff and visible state.
class WSClient {
  state: "closed" | "half_open" | "open" = "closed";
  private delay = 500;

  async connectWithBackoff(): Promise<void> {
    let consecutive = 0;
    while (!this.stopped) {
      try {
        await this.openSocket();
        this.state = "closed";
        this.delay = 500;
        return;
      } catch (err) {
        consecutive += 1;
        if (consecutive >= 5) this.state = "open";
        await sleep(Math.min(this.delay, 30_000));
        this.delay = Math.min(this.delay * 2, 30_000);
      }
    }
  }
}
```

## Rule R-7: Backpressure handled with bounded queues

Every consumer has a bounded ingest queue. On overflow:

- Drop oldest (or newest — project chooses) with reason
  `queue_overflow`.
- Increment drop counter.
- Expose queue depth as a gauge metric.

Unbounded `asyncio.Queue()` / `make(chan T)` (unbuffered) used as
ingest = **MAJOR**. Unbounded + no drop metric = **BLOCKER** (will
OOM under traffic spike).

### Python

```python
queue: asyncio.Queue[Event] = asyncio.Queue(maxsize=10_000)

async def ingest(ev: Event) -> None:
    try:
        queue.put_nowait(ev)
    except asyncio.QueueFull:
        DROPPED_MESSAGES.labels(reason="queue_overflow").inc()
        logger.warning("dropped on backpressure")
```

### Go

```go
// Bounded buffered channel — Go's idiomatic ingest queue.
ingest := make(chan Event, 10_000)

func enqueue(ev Event) {
    select {
    case ingest <- ev:
        ingestDepth.Set(float64(len(ingest)))
    default:
        droppedMessages.WithLabelValues("queue_overflow").Inc()
        slog.Warn("dropped on backpressure")
    }
}
```

## Rule R-8: Dead Letter Queue policy is explicit

For messages that look valid enough to be worth keeping but cannot be
processed right now (downstream DB down, feature-flag off), a DLQ is
appropriate. Rules:

1. DLQ policy MUST be written in the architecture doc — size limit,
   retention, replay procedure.
2. DLQ depth is exposed as a metric.
3. A DLQ that grows unboundedly and is never drained = **MAJOR**.
4. DLQ never receives unparseable JSON — that is a drop, not a DLQ
   entry. Unparseable messages are dead on arrival.

### Python / Go / TS

The shape of the DLQ varies (NATS JetStream DLQ subject, Kafka
deadletter topic, in-process bounded slice). The rule is independent
of the storage choice: **policy in docs + depth metric + drainage
procedure + no-DOA-bytes**. Reviewers cite R-8 with the rule text;
no canonical code snippet — the failure mode is *operational*, not
syntactic.

## Rule R-9: Timestamp discipline

External clocks lie. Every message carries either its own timestamp
(from the source) or arrives with a broker timestamp.

1. **Store both.** `source_ts` and `received_ts` are separate
   columns. Skew analysis becomes trivial later.
2. **Timezone:** all stored timestamps are UTC. Naive `datetime.now()`
   / `time.Now()` (no UTC) anywhere near data ingestion = **MAJOR**.
3. **Drift guard:** a message whose `source_ts` is > N minutes in the
   future (default N = 10) is dropped with reason `timestamp_drift`.
   Clock skew from a misconfigured device must not poison time-windowed
   aggregations.
4. **No reliance on `source_ts` for ordering** unless the source is
   known-good (GPS-disciplined clock, atomic time) — use
   `received_ts`.

### Python

```python
received_ts = datetime.now(timezone.utc)
if msg.source_ts > received_ts + timedelta(minutes=10):
    DROPPED_MESSAGES.labels(reason="timestamp_drift").inc()
    return
```

### Go

```go
receivedTs := time.Now().UTC()
if msg.SourceTs.After(receivedTs.Add(10 * time.Minute)) {
    droppedMessages.WithLabelValues("timestamp_drift").Inc()
    return
}
```

### TypeScript

```ts
const receivedTs = new Date(); // UTC milliseconds since epoch
if (msg.sourceTs.getTime() > receivedTs.getTime() + 10 * 60_000) {
  metrics.dropped.inc({ reason: "timestamp_drift" });
  return;
}
```

## Rule R-10: Observable counters are mandatory

Every consumer exposes at minimum:

```
# Counters
messages_received_total{source="..."}
messages_dropped_total{source="...", reason="..."}
messages_accepted_total{source="..."}
messages_dlq_total{source="..."}

# Gauges
ingest_queue_depth{source="..."}
broker_connection_state{source="..."}  # 0=closed, 1=half, 2=open
last_successful_receive_ts{source="..."}

# Histograms
message_processing_seconds{source="...", outcome="..."}
```

A consumer without these counters is operationally blind. On-call
cannot distinguish "broker is down" from "messages silently dropped".
Missing metrics = **MAJOR**; absent `messages_dropped_total{reason}`
specifically = **BLOCKER**.

In Go: register with `promauto.NewCounterVec` (or the project's
`internal/metrics` wrapper). In Python: `prometheus_client`. In TS
(Node): `prom-client`. Browser clients that consume push channels
expose drop counters via `/api/client-metrics` POST so the backend
can aggregate.

## Rule R-11: No business logic in the drop path

A dropped message must NOT trigger side effects downstream:

- Do not write a partial row to the analytics DB "just in case".
- Do not emit a downstream event with best-guess fields.
- Do not call an external API with a placeholder payload.

We chose to drop because we had no valid business interpretation;
inventing one poisons dashboards. Violations = **BLOCKER**.

This rule is hardest to spot in code review — it requires reading the
drop branch and checking what it touches. Reviewers cite R-11 when
the drop branch contains anything beyond `metric.inc + log.Warn +
return`.

## Rule R-12: Test the failure path

Every consumer has tests for:

- Malformed JSON / proto → drop + counter increments + consumer alive.
- Missing required field → drop with `missing_required_tag:*`.
- Renamed-tag via alias → accepted.
- Circuit breaker opens on N consecutive failures.
- Queue overflow drops oldest + counter.
- Health / readiness reflects drop rate at thresholds.

A consumer without failure-path tests = **MAJOR**. Without any
failure tests = **BLOCKER**. In Go, `testcontainers-go` for the
backing broker / DB; in Python, `pytest-asyncio` with a fake broker
client; in TS (browser), unit-level tests of the parsing layer plus
Playwright at the integration boundary.
